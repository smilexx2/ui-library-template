# CustomButton

The `CustomButton` component extends the standard button with enhanced visual effects, animations, and additional features while maintaining full compatibility with shadcn/ui design patterns.

- üéØ **Icon Support**: Left and right icon slots with automatic sizing

## Installation

```bash
npm install @smilexx2/custom-button
```

## Basic Usage

```tsx
import { CustomButton } from "@smilexx2/custom-button"

function App() {
  return <CustomButton variant="default">Click me</CustomButton>
}
```

## API Reference

### Props

| Prop        | Type              | Default     | Description                                     |
| ----------- | ----------------- | ----------- | ----------------------------------------------- |
| `variant`   | `ButtonVariant`   | `"default"` | The visual style variant of the button          |
| `size`      | `ButtonSize`      | `"default"` | The size of the button                          |
| `animation` | `ButtonAnimation` | `"none"`    | Animation effect for the button                 |
| `loading`   | `boolean`         | `false`     | Shows a loading spinner and disables the button |
| `ripple`    | `boolean`         | `false`     | Enables ripple effect on click                  |
| `disabled`  | `boolean`         | `false`     | Disables the button                             |
| `leftIcon`  | `ReactNode`       | `undefined` | Icon to display on the left side of the button  |
| `rightIcon` | `ReactNode`       | `undefined` | Icon to display on the right side of the button |

### Type Definitions

```tsx
type ButtonVariant =
  | "default"
  | "destructive"
  | "outline"
  | "secondary"
  | "ghost"
  | "link"
  | "gradient"
  | "glow"
  | "neon"

type ButtonSize = "xs" | "sm" | "default" | "lg" | "xl" | "icon"

type ButtonAnimation = "none" | "pulse" | "bounce" | "wobble" | "ripple"
```

## Examples

### Form Submit Button

```tsx
<CustomButton variant="gradient" size="lg" loading={isSubmitting} onClick={handleSubmit}>
  {isSubmitting ? "Submitting..." : "Submit Form"}
</CustomButton>
```

### Icon Button

```tsx
<CustomButton size="icon" variant="ghost" onClick={() => setLiked(!liked)}>
  <Heart className={liked ? "fill-current" : ""} />
</CustomButton>
```

### Call to Action

```tsx
<CustomButton
  variant="glow"
  size="xl"
  animation="pulse"
  leftIcon={<Rocket />}
  onClick={handleGetStarted}
>
  Get Started
</CustomButton>
```

## Accessibility

The CustomButton component follows WAI-ARIA best practices:

- Proper `button` role
- Keyboard navigation support (Space/Enter to activate)
- Focus indicators
- Disabled state handling
- Loading state announcements
- ARIA labels support

## Styling

The component uses Tailwind CSS v4 and CSS custom properties for theming. All styles are automatically included when you import the component.

### Customizing Colors

You can customize the button colors using CSS variables in your global styles:

```css
:root {
  --gradient-from: rgb(59, 130, 246);
  --gradient-to: rgb(147, 51, 234);
  --glow-color: rgb(59, 130, 246);
  --neon-color: rgb(34, 211, 238);
}
```

## Migration from Standard Button

Migrating from shadcn/ui Button to CustomButton is straightforward:

```tsx
// Before
import { Button } from "@/components/ui/button"
;<Button variant="default">Click me</Button>

// After
import { CustomButton } from "@smilexx2/custom-button"
;<CustomButton variant="default">Click me</CustomButton>
```

All existing props are supported, with additional features available when needed.
