# Installation

Get started with @smilexx2 UI Library in your React project.

## Prerequisites

Before installing, ensure your project meets these requirements:

- **Node.js** 18.0 or later
- **React** 18.0 or later (React 19 compatible)
- **React DOM** 18.0 or later

### Required if using shadcn/ui

This library extends shadcn/ui, so you'll need:
- shadcn/ui components installed
- Tailwind CSS configured
- Required shadcn/ui dependencies

## Package Installation

Our library is modular - install only what you need:

### Core Utilities

```bash
npm install @smilexx2/ui-core
```

Provides the `cn` utility function for className management.

### UI Components

```bash
npm install @smilexx2/ui
```

shadcn/ui compatible components with consistent styling.

### Custom Components

```bash
npm install @smilexx2/custom-button
```

Enhanced components with additional features.

### Install Everything

```bash
npm install @smilexx2/ui-core @smilexx2/ui @smilexx2/custom-button
```

## Tailwind CSS Configuration

### For Tailwind CSS v4 (Latest)

If you're using Tailwind CSS v4, add the library paths to your CSS:

```css
/* app.css or globals.css */
@import "tailwindcss";
@import "@smilexx2/ui/styles.css";
@import "@smilexx2/custom-button/styles.css";
```

### For Tailwind CSS v3

Add the library paths to your content configuration:

```js
// tailwind.config.js
module.exports = {
  content: [
    "./src/**/*.{js,ts,jsx,tsx,mdx}",
    "./node_modules/@smilexx2/ui/dist/**/*.{js,cjs}",
    "./node_modules/@smilexx2/custom-button/dist/**/*.{js,cjs}"
  ],
  theme: {
    extend: {},
  },
  plugins: [],
}
```

## Import Styles

Don't forget to import the component styles in your app:

```tsx
// In your root component or CSS file
import '@smilexx2/ui/styles.css'
import '@smilexx2/custom-button/styles.css'
```

## Troubleshooting

### Module Resolution Issues

If you encounter import errors:

1. Ensure you're using a modern bundler (Vite, Next.js 13+, etc.)
2. Check your `tsconfig.json` moduleResolution setting
3. Try clearing your node_modules and reinstalling

### Style Issues

If styles aren't applying:

1. Verify you've imported the CSS files
2. Check your Tailwind configuration includes the library paths
3. Ensure your bundler processes CSS imports

### TypeScript Errors

If you see type errors:

1. Update TypeScript to 5.0 or later
2. Check that all peer dependencies are installed
3. Restart your TypeScript server in your IDE
