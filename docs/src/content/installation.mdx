# Installation

Get started with @smilexx2 UI Library in your React project.

## Prerequisites

Before installing, ensure your project meets these requirements:

- **Node.js** 18.0 or later
- **React** 18.0 or later (React 19 compatible)
- **React DOM** 18.0 or later

### Optional shadcn/ui Integration

This library is designed to work seamlessly with shadcn/ui, but it's not required. The components work standalone. If you want shadcn/ui integration:
- Install shadcn/ui components as needed
- The libraries share the same design principles and will work together

## Package Installation

Our library is modular - install only what you need:

### Core Utilities

```bash
npm install @smilexx2/ui-core
```

Provides the `cn` utility function for className management.

### UI Components

```bash
npm install @smilexx2/ui
```

shadcn/ui compatible components with consistent styling.

### Custom Components

```bash
npm install @smilexx2/custom-button
```

Enhanced components with additional features.

### Install Everything

```bash
npm install @smilexx2/ui-core @smilexx2/ui @smilexx2/custom-button
```

## Optional: Tailwind CSS Integration

The component library works perfectly without Tailwind CSS - all components come with their own styles. However, if you want to use Tailwind utility classes for additional customization, you can optionally install and configure Tailwind CSS v4:

```css
/* app.css or globals.css - Only needed if you want Tailwind utilities */
@import "tailwindcss";
```

## Peer Dependencies

The library has minimal peer dependencies - only React is required:

```json
{
  "react": "^18.0.0 || ^19.0.0",
  "react-dom": "^18.0.0 || ^19.0.0"
}
```

**Note**: All other dependencies like `@radix-ui/react-slot`, `class-variance-authority`, `clsx`, and `tailwind-merge` are included as regular dependencies, so you don't need to install them separately.

## Troubleshooting

### Module Resolution Issues

If you encounter import errors:

1. Ensure you're using a modern bundler (Vite, Next.js 13+, etc.)
2. Check your `tsconfig.json` moduleResolution setting
3. Try clearing your node_modules and reinstalling

### Style Issues

If component styles aren't applying:

1. Verify your bundler supports CSS processing
2. Check that the component packages are properly installed
3. Ensure your build process includes the component files

### TypeScript Errors

If you see type errors:

1. Update TypeScript to 5.0 or later
2. Check that all peer dependencies are installed
3. Restart your TypeScript server in your IDE
